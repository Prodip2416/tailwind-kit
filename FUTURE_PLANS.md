# 🔭 TailwindKit — Future Upgrade Plans

> Strategic enhancements planned to evolve TailwindKit into a full-scale design system platform.

---

## ✅ Already Completed

- [x] **TypeScript Support** — JS/TS code generation with live toggle across all 8 menus
- [x] **Two-Panel Layout** — Side-by-side live demo + code viewer for every component
- [x] **Dark Mode** — System-aware dark/light theme with smooth transitions
- [x] **Drag & Drop** — Sortable lists, Kanban board, file drop zone, card reordering
- [x] **Responsive Design** — Mobile-first, tested across all breakpoints

---

## 🧪 Phase 1 — Developer Experience (Q3 2026)

| Feature | Description | Impact |
|---|---|---|
| **Storybook Integration** | Isolated component stories with controls, a11y checks, and auto-generated docs | High |
| **Vitest + Testing Library** | Unit & interaction tests for all 80+ components | High |
| **Component CLI Generator** | `npx tailwindkit add Button` scaffolds component with boilerplate | High |
| **ESLint + Prettier strict config** | Enforce consistent code quality standards across codebase | Medium |

**Why it matters:** Recruiters and open-source contributors expect a mature DX. Test coverage and a CLI signal production-grade engineering.

## 🎨 Phase 2 — Design System Maturity (Q4 2026)

| Feature | Description | Impact |
|---|---|---|
| **Theme Builder UI** | Visual editor to customize colors, radius, spacing — exports Tailwind config | Very High |
| **Design Tokens / CSS Variables** | Centralized token system for colors, typography, spacing | High |

## ⚡ Phase 3 — Performance & Scale (Q1 2027)

| Feature | Description | Impact |
|---|---|---|
| **Tree-shakeable NPM Package** | Publish as `@tailwindkit/ui` — import only what you use, zero bloat | Very High |
| **Bundle Size Optimization** | Code splitting + lazy loading per category | High |
| **SSR / Next.js Compatibility** | Server-side rendering support for Next.js and Remix | High |
| **Lighthouse Score Tracking** | Automated performance + a11y audits on every PR via CI | Medium |

**Why it matters:** Shipping as an npm package is the leap from "demo project" to "real library." This is the milestone that changes everything on a resume.
